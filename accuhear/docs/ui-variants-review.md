# Omni UI Variants Review

Scenario used across all five variants: the same dense patient follow-up workspace for Eleanor Pena, including the existing left-nav app shell, top utility bar, quick actions, clinical form controls, operational tasks, fulfillment/order data, and recent activity.

## What each variant is testing

- Variant 01 tests a neutral enterprise compact model with familiar shadcn-like cards, clear hierarchy, and balanced two-column density.
- Variant 02 tests a warm clinical dense model that keeps compactness but softens the environment for repeated patient-facing workflow use.
- Variant 03 tests a sharp operations console model with stronger command surfaces, contrast, and queue-pressure visibility.
- Variant 04 tests a premium compact workspace model that uses refined split panels and a polished utility rail without becoming airy.
- Variant 05 tests an extreme high-density power-user mode that pushes the most actionable content above the fold and treats the page more like a control board than a card stack.

## Density tradeoffs and utility gains

- Variant 01 trades some maximum density for very fast learnability. It improves utility over the current UI by tightening headers, reducing decorative spacing, and consolidating tasks, charting, and fulfillment into one glanceable layout.
- Variant 02 trades a little raw information volume for lower visual fatigue. It improves utility by making charting and checkout read more like a clinical packet while still keeping action surfaces compact.
- Variant 03 trades warmth for speed under pressure. It improves utility by making next actions, queue state, and financial blockers more obvious than in the current softer shell.
- Variant 04 trades a small amount of raw density for a more deliberate split-panel workflow. It improves utility by grouping task modes into a clearer rail structure that could scale well across patient tabs.
- Variant 05 trades comfort and onboarding ease for maximum operator throughput. It improves utility by collapsing nearly all critical workflow elements into tables and compact grids with minimal vertical waste.

## Recommendation

- Strongest foundation: Variant 01. It is the safest upgrade path from the current Omni shell while still materially improving density, scan speed, and practical utility.
- Strongest stretch direction: Variant 03. It has the clearest operational posture for dense clinical work and could produce the best speed gains if the team is comfortable with a more assertive console-like interface.
- Variant 05 is the best reference for a future optional “power mode,” but it is probably too aggressive to use as the default style guide baseline.
