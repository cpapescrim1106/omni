## Codebase Patterns
- Prisma schema is source-of-truth at `prisma/schema.prisma`; run `npx prisma validate` after schema edits.
- Story-level schema migrations are stored in `prisma/migrations/<timestamp>_<name>/migration.sql`.
- Seed normalization helpers live in `scripts/seed.ts`; baseline statuses should be centralized there and reused by tests.
- In-clinic appointment state changes should flow through `src/lib/appointments/status-transition.ts` so guard rules, lifecycle timestamps, and `AppointmentStatusEvent` audit writes remain consistent.
- Monitor feed filtering + timing derivation should live in `src/lib/appointments/monitor-feed.ts`; API routes should stay thin wrappers around that query service.
- Entry-point-specific status actions should route through `src/lib/appointments/status-action-handlers.ts` so schedule-context and monitor-list handlers enforce the same transition availability matrix.
- Keep monitor card semantics (active filtering, status color tones, wait/in-progress timer math) in a shared helper (`src/lib/appointments/monitor-ui.ts`) so React rendering and tests stay deterministic.
